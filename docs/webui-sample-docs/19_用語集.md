## 技術用語集

| 用語 | 定義 |
|------|------|
| アクセストークン | APIリクエストの認証に使用する短期有効なJWT |
| インクリメンタルバックアップ | 前回のバックアップ以降に変更されたデータのみをバックアップする方式 |
| エラーバウンダリ | Reactコンポーネントのエラーをキャッチして代替UIを表示する仕組み |
| コードスプリッティング | JavaScriptバンドルを複数のファイルに分割して初期ロードを高速化する手法 |
| コンテナ | アプリケーションとその依存関係をパッケージ化した実行環境（Docker） |
| コンポーネント | 再利用可能なUI部品（React） |
| サービスワーカー | バックグラウンドで動作するWebブラウザのスクリプト（PWA等に使用） |
| セッション | ユーザーのログイン状態を維持する仕組み |
| ソフトデリート | データを物理的に削除せず、削除フラグを立てて論理的に非表示にする手法 |
| 差分バックアップ | 最後のフルバックアップ以降に変更されたデータをバックアップする方式 |
| タイムゾーン | 地域ごとの時刻差（UTC+9 = 日本標準時） |
| デバウンス | 連続したイベント（入力等）の処理を一定時間後に一度だけ実行する最適化手法 |
| ハッシュ | データを固定長の値に変換する一方向関数（パスワード保存に使用） |
| パーティション | データベーステーブルを分割する手法（パフォーマンス向上） |
| フォールバック | 主要な方法が使えない場合の代替手段 |
| フルバックアップ | すべてのデータを完全にバックアップする方式 |
| プロキシ | クライアントとサーバーの間に介在してリクエストを中継するサーバー |
| ページネーション | 大量のデータを複数ページに分割して表示する手法 |
| マイグレーション | データベーススキーマの変更を管理・適用する仕組み |
| リフレッシュトークン | アクセストークンの更新に使用する長期有効なトークン |
| リポジトリパターン | データアクセスロジックを抽象化するデザインパターン |
| ロールバック | システムやデータベースを以前の状態に戻す操作 |

## バックアップ関連用語

| 用語 | 定義 |
|------|------|
| RPO（Recovery Point Objective） | 復旧目標時点。障害発生時に許容できるデータ損失の最大時間 |
| RTO（Recovery Time Objective） | 復旧目標時間。サービスを復旧するまでの目標時間 |
| 保持ポリシー | バックアップデータを保存しておく期間・世代数に関するルール |
| 暗号化バックアップ | バックアップデータを暗号化して保存する手法（AES-256等） |
| 復元ポイント | バックアップからデータを復元できる特定の時点 |
| ストレージプール | バックアップデータの保存先として管理されたストレージリソースの集合 |
