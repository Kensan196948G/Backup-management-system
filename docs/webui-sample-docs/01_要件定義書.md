# 要件定義書

**プロジェクト名:** バックアップ管理システム WebUI
**バージョン:** 1.0.0
**作成日:** 2026-02-25
**ステータス:** ドラフト

---

## 1. はじめに

### 1.1 目的

本文書は、バックアップ管理システムの Web ユーザーインターフェース（WebUI）に関する要件を定義します。システム管理者および運用担当者が、バックアップジョブの管理・監視・レポートを Web ブラウザから直感的に操作できることを目的とします。

### 1.2 対象読者

- プロジェクトマネージャー
- システムアーキテクト
- フロントエンド / バックエンド開発者
- QA エンジニア
- システム管理者（ステークホルダー）

### 1.3 スコープ

- バックアップジョブの作成・編集・削除・実行
- バックアップ状態のリアルタイム監視ダッシュボード
- レポート生成・エクスポート機能
- ユーザー管理・ロールベースアクセス制御（RBAC）
- 通知設定（Email / Microsoft Teams）

## 2. 機能要件

### 2.1 ダッシュボード

| ID | 要件 | 優先度 |
|----|------|--------|
| F-001 | バックアップジョブの成功/失敗/実行中のサマリーをカード形式で表示する | Must |
| F-002 | 直近 24 時間のバックアップ履歴をタイムラインで表示する | Must |
| F-003 | ストレージ使用率をグラフで表示する | Should |
| F-004 | ウィジェットのカスタマイズ（表示順序・追加・削除）を可能にする | Could |

### 2.2 バックアップジョブ管理

| ID | 要件 | 優先度 |
|----|------|--------|
| F-010 | バックアップジョブの一覧表示（ページネーション・ソート・フィルタ付き） | Must |
| F-011 | ジョブの新規作成ウィザード（ステップ形式） | Must |
| F-012 | ジョブの即時実行・スケジュール設定 | Must |
| F-013 | ジョブの有効化・無効化トグル | Must |
| F-014 | バッチ操作（複数ジョブの一括実行・削除） | Should |
| F-015 | ジョブのクローン（既存設定をコピーして新規作成） | Could |

### 2.3 監視・アラート

| ID | 要件 | 優先度 |
|----|------|--------|
| F-020 | ジョブ実行中のリアルタイム進捗バー表示 | Must |
| F-021 | 失敗ジョブのアラート表示（バナー・バッジ） | Must |
| F-022 | アラート通知ルールの設定（条件・通知先） | Must |
| F-023 | アラート履歴の表示・検索 | Should |

### 2.4 レポート

| ID | 要件 | 優先度 |
|----|------|--------|
| F-030 | 日次・週次・月次のバックアップサマリーレポート生成 | Must |

### 2.5 ユーザー管理

| ID | 要件 | 優先度 |

| F-043 | 多要素認証（MFA）サポート | Should |

## 3. 非機能要件

### 3.1 パフォーマンス

| ID | 要件 | 目標値 |
|----|------|--------|
| NF-001 | ページ初回表示（First Contentful Paint） | 2 秒以内 |
| NF-002 | API レスポンスタイム（95 パーセンタイル） | 500ms 以内 |
| NF-003 | 同時接続ユーザー数 | 100 ユーザー以上 |
| NF-004 | ダッシュボードのリアルタイム更新間隔 | 30 秒 |

### 3.2 可用性

| ID | 要件 | 目標値 |
|----|------|--------|
| NF-010 | システム稼働率（SLA） | 99.5% 以上（月間） |
| NF-011 | 計画メンテナンス通知 | 48 時間前 |
| NF-012 | 障害時の自動フェイルオーバー | 60 秒以内 |

### 3.3 セキュリティ

| ID | 要件 |
|----|------|
| NF-020 | 全通信は HTTPS（TLS 1.2 以上）を使用する |
| NF-021 | パスワードは bcrypt（コスト係数 12 以上）でハッシュ化する |
| NF-022 | SQL インジェクション・XSS・CSRF 対策を実施する |
| NF-023 | 操作ログ（監査ログ）を 1 年間保存する |

### 3.4 ユーザビリティ

| ID | 要件 |
|----|------|
| NF-030 | WCAG 2.1 AA 準拠のアクセシビリティ |
| NF-031 | レスポンシブデザイン（1280px 以上、タブレット対応） |
| NF-032 | 日本語・英語の UI 切り替えサポート |
| NF-033 | ダークモード / ライトモードの切り替え |

### 3.5 互換性

| ブラウザ | 最小バージョン |
|---------|--------------|
| Chrome | 最新 -2 |
| Firefox | 最新 -2 |
| Edge | 最新 -2 |
| Safari | 15 以上 |

## 4. 制約条件

- バックエンド API は既存の REST API（FastAPI）を使用する
- データベースは PostgreSQL 15 以上を使用する
- フロントエンドフレームワークは React 18 以上を採用する
- コンテナ化（Docker）による提供が必須

---

## 5. 前提条件・依存関係

- バックアップソフトウェア（Veeam / AOMEI）との連携 API が利用可能であること
- CI/CD パイプライン（GitHub Actions）が構築済みであること
- SSL 証明書の発行・更新体制が整備されていること

---

## 6. 受入基準

- 全 Must 要件の実装・テスト完了
- パフォーマンス要件 NF-001 ～ NF-004 を実測で満たすこと
- セキュリティ脆弱性診断（OWASP Top 10）で重大・高リスクゼロ
- ユーザー受入テスト（UAT）の合格率 95% 以上
