# 並列開発コマンド

複数のタスクを並列で実行して開発効率を最大化します。

## 🚀 並列実行の概念

このコマンドは、独立した複数のタスクを同時に実行することで、開発時間を大幅に短縮します。

## 📋 使用可能な並列実行パターン

### パターン1: 情報収集の並列化

複数の情報源から同時にデータを収集します：

```bash
# 以下を並列実行
git status
git diff --stat
git log --oneline -10
git branch -a
```

### パターン2: コード分析の並列化

複数のコード分析ツールを同時に実行します：

```bash
# 以下を並列実行
flake8 . --statistics
black --check .
isort --check-only .
mypy . --ignore-missing-imports
```

### パターン3: テストの並列化

複数のテストカテゴリを同時に実行します：

```bash
# 以下を並列実行
pytest tests/unit/ -v
pytest tests/integration/ -v
pytest tests/functional/ -v
```

## 🎯 並列実行の利点

1. **時間短縮**: 複数タスクを同時実行で待ち時間を削減
2. **効率向上**: リソースを最大限活用
3. **スループット向上**: より多くのタスクを短時間で完了

## ⚙️ 実行ガイドライン

### 並列実行可能な条件

✅ **並列実行できる場合:**
- タスク間に依存関係がない
- 異なるファイルを操作する
- 読み取り専用の操作（git status, diff等）
- 独立したテストケース

❌ **並列実行できない場合:**
- タスク間に依存関係がある（AがBの結果を必要とする）
- 同じファイルを編集する
- データベースのマイグレーション
- 順序が重要な操作

## 📖 使用方法

ユーザーが `/parallel-dev` と入力した場合、以下のように並列実行可能なタスクを提案してください：

1. 現在のコンテキストを分析
2. 並列実行可能なタスクを特定
3. タスクを同時に実行
4. 結果を統合して報告

## ⚠️ 注意事項

1. **リソース管理**: CPU/メモリの使用状況に注意
2. **エラーハンドリング**: 一部が失敗しても他は継続
3. **順序の保証**: 並列実行では順序は保証されない
4. **競合の回避**: 同じリソースへの同時アクセスを避ける
