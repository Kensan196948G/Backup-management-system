# コードベース探索コマンド

Explore Agentを使用して、コードベースを効率的に探索します。

## 📋 概要

このコマンドは、Explore SubAgentを起動してコードベースの構造や特定の機能の実装場所を調査します。

## 🎯 使用目的

- 新しいコードベースの理解
- 特定の機能の実装場所の特定
- アーキテクチャの理解
- 依存関係の調査
- コードパターンの発見

## 🔍 探索レベル

### Quick（高速）
- 基本的な検索
- ファイル名やディレクトリ構造の確認
- 主要なエントリポイントの特定

### Medium（中程度）
- 複数の場所を調査
- 関連ファイルの探索
- 基本的なコード分析

### Very Thorough（徹底的）
- 包括的な分析
- 複数の命名規則を確認
- 詳細な依存関係の追跡

## 📝 使用例

### 例1: 認証機能の探索

```
/explore

認証機能の実装場所を調査してください。
探索レベル: medium
```

### 例2: API エンドポイントの探索

```
/explore

すべてのAPIエンドポイントを見つけて、その構造を説明してください。
探索レベル: very thorough
```

### 例3: データベースモデルの探索

```
/explore

データベースモデルの定義場所と、それらの関係性を調査してください。
探索レベル: medium
```

## ⚙️ 実行フロー

1. **探索範囲の特定**: ユーザーの要求から探索対象を特定
2. **Agent起動**: 適切な探索レベルでExplore Agentを起動
3. **結果の分析**: Agentが返した情報を分析
4. **レポート作成**: わかりやすい形式で結果を報告

## 💡 ヒント

- 探索レベルは、時間とコストのトレードオフを考慮して選択
- 具体的な質問ほど、効率的な探索が可能
- 探索結果をメモリに保存して、後で参照可能
